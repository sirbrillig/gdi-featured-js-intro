Functions

Functions are separable, reusable pieces of code.

Photo credit: Rick Payette cc

Real Function Example

1. Write instructions on a piece of paper, describing how to fetch a pencil. 2. Show the paper to your neighbor and ask them to follow the instructions. 3. Pass the paper to someone else. 4. The new owner asks their neighbor to follow the instructions. --- The paper is a function and its instructions can be followed multiple times by different people.

Declaring Functions

To declare (create) a function, you can give it a name, then include all the code for the function inside curly brackets {}

function printParrotFacts() {
  console.log('Some parrot species can live for over 80 years');
  console.log('Kakapos are a critically endangered flightless parrot');
}

Functions can have multiple lines.

Using Functions

To call (execute) a function, you type its name, followed by parenthesis ()

 printParrotFacts();

We'll talk about what can go inside those parenthesis in a minute! For now, leave them empty.

What's going on here?

A function is a block of code you can reuse many times. Whenever you call a function by using its name, you tell the browser to run the code inside the function.

You must declare a function before you can use it.

Functions in Variables

Most functions have a variable pointing to them with a descriptive name, just like any other value. (If there is no name, it's called an "anonymous" function.)

Real Argument Example

1. Change instructions on paper to explain how to fetch anything (call it `THING`). 2. On another piece of paper write `THING = ` followed by something to fetch. 2. Show both papers to your neighbor and ask them to follow the instructions, replacing `THING` with whatever the other paper says. --- The second paper is a function "argument". It's a variable!

Arguments

Functions can accept input values, called arguments.

function callKitten(kittenName){
  console.log('Come here, ' + kittenName + '!');
}

callKitten('Fluffy'); // outputs 'Come here, Fluffy!'

Multiple Arguments

Multiple arguments can be separated by commas.

function addNumbers(a, b){
  console.log(a + b);
}

addNumbers(5, 7);  // outputs 12
addNumbers(9, 12); // outputs 21

Variables as Arguments

Because variables contain values, you can also pass variables into functions. These variables do not need to have the same name as the function arguments.

function addOne(num) {
  let newNumber = num + 1;
  console.log('You now have ' + newNumber);
}

// Declare variables
let numberOfKittens = 5;
let numberOfPuppies = 4;

// Use them in functions
addOne(numberOfKittens);
addOne(numberOfPuppies);

Real Returning Example

1. Write down instructions on paper to return the square of a number. 2. Ask your neighbor to execute the instructions and write down the result on another piece of paper. 3. When they are done, have them give the other paper back to you. --- The other paper (the result) is a function's **"return value"**.

Returning Values

You can have a function give you back a value, to use later.

function square(num) {
  return num * num;
}

console.log(square(4));       // outputs '16'

let squareOfFive = square(5); // squareOfFive equals '25'

return will immediately end a function.

Anonymous Functions

Functions don't have to have a name, but you need to put them in a variable to use them. ```js const sayHi = function(name) { console.log(name + ' says hi.'); }; sayHi('Maria'); // prints "Maria says hi." ```
### Fat-arrow Functions Anonymous functions can also be defined using a shorthand with the "fat arrow" `=>` _after_ the parenthesis instead of using the word `function`. ```js const sayHi = (name) => { console.log(name + ' says hi.'); }; sayHi('Maria'); // prints "Maria says hi." ```
### One-line Functions Short fat-arrow functions can be written on one line more easily. They return the result of the last statement. ```js const getGreeting = (name) => 'Hi ' + name; console.log(getGreeting('Fuzzy')); // prints "Hi Fuzzy" ```

Variable Scope

The scope of a variable determines where it's value is accessible throughout your program.

Photo credit: _vikram cc

Global Scope

A variable declared outside of a function has a global scope and can be accessed anywhere, even inside of functions.

let awesomeGroup = 'Girl Develop It'; // Global scope

function whatIsAwesome() {
  console.log(awesomeGroup + ' is pretty awesome.'); // Will work
}

whatIsAwesome();

Local Scope

A variable declared inside a function has a local scope and can only be accessed within that function.

function whatIsAwesome() {
  let awesomeGroup = 'Girl Develop It';              // Local scope
  console.log(awesomeGroup + ' is pretty awesome.'); // Will work
}

whatIsAwesome();

console.log(awesomeGroup + ' is pretty awesome.'); // Won't work

Boolean Variables

Photo credit: elycefeliz cc

Boolean Variables

Boolean variables represent the logical values true and false

let catsAreBest = true;
let dogsRule = false;

Coercion to Boolean

When using coercion to change a non-Boolean variable to a Boolean, does it become `true` or `false`? ```js let numberOfPuppies = 4; console.log(Boolean(numberOfPuppies)); // prints 'true' ```

Truthy and Falsy

The following are considered **"falsy"** in JavaScript because they coerce to Boolean `false`: - `""` (empty string) - `0`, `-0`, `NaN` (invalid number) - `null`, `undefined` --- Any other value is considered **"truthy"**.

Control Flow

Photo credit: DennisM2 cc

The if statement

Use if statements to decide which lines of code to execute, based on a condition.

if (condition) {
  // statements to execute
}

let age = 30;

if (age > 18) {
  console.log('You are an adult');
}

Conditions

Conditions must always be a Boolean or they will coerced into a Boolean (true or false).

let age = 30;

if (age > 18) {
  console.log('You are an adult');
}

if (age) {
  console.log('You are alive');
}

Comparison Operators

Example Name Result
a == b Equal TRUE if a is equal to b (can be different types).
a === b Identical TRUE if a is equal to b, and the same type.
a != b Not equal TRUE if a is not equal to b (can be different types).
a !== b Not identical TRUE if a is not equal to b, or they are not the same type.
a < b Less than TRUE if a is strictly less than b.
a > b Greater than TRUE if a is strictly greater than b.
a <= b Less than or equal to TRUE if a is less than or equal to b.
a >= b Greater than or equal to TRUE if a is greater than or equal to b.

Watch out!

Don't mix up = and == and ===

The if/else statement

Use else to provide an alternate set of instructions.

let age = 30;

if (age >= 16) {
  console.log('Yay, you can drive!');
} else {
  console.log('Sorry, you have ' + (16 - age) +
  ' years until you can drive.');
}

The if/else statement

If you have multiple conditions, you can use else if.

let age = 30;

if (age >= 35) {
  console.log('You can vote AND run for President!');
} else if (age >= 30) {
  console.log('You can vote AND run for the Senate!');
} else if (age >= 18) {
  console.log('You can vote!');
} else {
  console.log('You can\'t vote, but you can write your representatives.');
}

Logical Operators

Example Name Result
a && b And TRUE if both a and b are TRUE.
a || b Or TRUE if either a or b is TRUE.
!a Not TRUE if a is not TRUE.

Using logical operators

You can use these operators to combine conditions.

let age = 30;
let yearsAsCitizen = 30;

if (age >=30 && yearsAsCitizen > 9) {
  console.log('You can run for the Senate!');
} else {
  console.log('You are not eligible to run for the Senate');
}
